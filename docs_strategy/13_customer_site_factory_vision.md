# 📑 Strategia: ZEUS Customer Site Factory (v1.0)

**Status:** Approvato per l'implementazione Backend

## 1. Visione d'Insieme
Trasformare ZEUS OS da una singola applicazione a una **Piattaforma di Generazione (Factory)**. ZEUS agirà come un "Architect-as-a-Service", capace di progettare, distribuire e gestire e-commerce e siti web per clienti terzi partendo da semplici prompt in linguaggio naturale.

## 2. Pilastri Tecnologici (The "Epic" Stack)
Per garantire che i siti dei clienti siano di livello enterprise, integriamo le seguenti logiche:

* **mcpEpic (Architectural Layer):** Prima di scrivere codice, ZEUS genera un Blueprint (PRD, Schema DB, API Spec). Questo evita "allucinazioni" e garantisce che il sito del cliente sia strutturalmente perfetto.
* **MCP Stitch (Data Mapping):** Utilizziamo il protocollo MCP per "cucire" i dati reali del cliente (prodotti, prezzi, inventario) sui componenti UI standardizzati di ZEUS.
* **Amber Glow UI Engine:** Tutti i siti generati ereditano il DNA estetico di ZEUS (Glassmorphism, Tailwind dinamico), garantendo una qualità visiva superiore.
* **Supabase Vault (Security):** Le chiavi API di Stripe o altri gateway dei clienti vengono salvate in modo criptato e isolate dal resto del sistema.

## 3. Workflow Utente (Customer Journey)
1.  **Intervista AI:** L'utente descrive il suo business nella Dashboard Cliente.
2.  **Generazione Blueprint:** Antigravity crea la struttura invisibile (il "cervello" del sito).
3.  **Visual Assembly:** ZEUS monta l'interfaccia usando i componenti pre-approvati.
4.  **Launch:** Puntamento del dominio e deploy automatico.

## 4. Manutenzione Agentica
Il cliente non usa un pannello di controllo complicato. Per modificare il sito, invia un comando a ZEUS:
> *"ZEUS, metti in saldo del 30% tutte le scarpe rosse fino a domenica."*
L'agente aggiorna il database e riflette la modifica sulla UI in tempo reale.

## 5. Tokenomics (Revenue Model)
* **Creazione:** Costo fisso in TND (o Token) per il primo deploy.
* **Gestione:** Consumo di NPUs (Neural Processing Units) per ogni modifica o interazione dell'agente con il sito del cliente.
